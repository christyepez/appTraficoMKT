# Arquitectura C4

Este documento describe la aplicacion en 4 niveles C4: contexto, contenedores, componentes y codigo.

## Nivel 3: Componentes

```mermaid
C4Component
title App Trafico MKT - Componentes principales

Container_Boundary(frontend, "Frontend Next.js") {
  Component(nav, "AppNav", "React", "Menu por permisos, marca institucional e idioma.")
  Component(reqPage, "Requerimientos", "React Page", "CRUD, tracking y cambios de estado.")
  Component(prodPage, "Productos", "React Page", "CRUD, adjuntos, aprobaciones y versiones enviadas.")
  Component(evidencePage, "Adjuntos", "React Page", "Cabecera-detalle de evidencias, preview y eliminacion logica.")
  Component(approvalPage, "Aprobaciones", "React Page", "Revision de producto, adjuntos y decision.")
  Component(auditPage, "Auditorias", "React Page", "Tracking consolidado de requerimientos, productos y aprobaciones.")
  Component(adminPages, "Administracion", "React Pages", "Catalogos, usuarios, marca, almacenamiento, notificaciones y carga inicial.")
}

Container_Boundary(backend, "Microservicios .NET") {
  Component(identityComp, "Identity endpoints", "Minimal API", "Auth local, SSO, usuarios, roles y marca.")
  Component(reqComp, "Requirement workflow", "Minimal API + Domain", "Estados, auditoria y metricas de requerimientos.")
  Component(actComp, "Activity workflow", "Minimal API + Domain", "Estados de productos, aprobaciones, notificaciones y sync con requerimiento.")
  Component(evidenceComp, "Evidence workflow", "Minimal API", "Archivos, storage local/cloud, aprobaciones y auditoria.")
  Component(adminComp, "Catalog/import workflow", "Minimal API", "Catalogos, relaciones FK y carga inicial Excel.")
  Component(domain, "BuildingBlocks", "C# Domain", "Entidades, enums, reglas de workflow y catalogos base.")
}

Rel(nav, identityComp, "Consulta marca/permisos")
Rel(reqPage, reqComp, "Gestiona requerimientos")
Rel(prodPage, actComp, "Gestiona productos")
Rel(prodPage, evidenceComp, "Carga y consulta adjuntos")
Rel(approvalPage, actComp, "Aprueba/rechaza productos")
Rel(approvalPage, evidenceComp, "Consulta adjuntos/aprobaciones")
Rel(auditPage, reqComp, "Lee auditoria requerimientos")
Rel(auditPage, actComp, "Lee auditoria productos")
Rel(auditPage, evidenceComp, "Lee auditoria aprobaciones")
Rel(adminPages, adminComp, "Gestiona catalogos y carga inicial")
Rel(identityComp, domain, "Usa roles/permisos")
Rel(reqComp, domain, "Usa reglas de requerimiento")
Rel(actComp, domain, "Usa reglas de producto")
```

## Nivel 4: Codigo

```mermaid
classDiagram
direction LR

class Requirement {
  +Guid Id
  +string Code
  +string ActivityOrEvent
  +RequirementStatus Status
  +StartAnalysis()
  +StartExecution()
  +Complete(total, approved)
  +Delete(user)
}

class TechnicalActivity {
  +Guid Id
  +Guid RequirementId
  +string ProductId
  +ActivityStatus Status
  +Start()
  +MarkEvidenceAttached()
  +SendToApproval()
  +Decide(decision)
  +Delete(user)
}

class EvidenceItem {
  +Guid Id
  +Guid ActivityId
  +string FileName
  +string StorageUrl
  +Delete(user)
}

class ActivityApproval {
  +Guid Id
  +Guid ActivityId
  +ApprovalDecision Decision
  +string ApprovedBy
  +string Comments
}

class AppUser {
  +Guid Id
  +string Email
  +string Roles
  +string ScreenPermissions
  +string MenuMode
  +bool IsActive
}

class CatalogReference {
  +Guid Id
  +string Type
  +string Code
  +string Name
}

class AuditEvent {
  +Guid Id
  +Guid EntityId
  +string FromStatus
  +string ToStatus
  +string Action
  +string PerformedBy
  +DateTimeOffset OccurredAt
}

Requirement "1" --> "0..*" TechnicalActivity : agrupa
TechnicalActivity "1" --> "0..*" EvidenceItem : adjuntos
TechnicalActivity "1" --> "0..*" ActivityApproval : versiones/aprobaciones
Requirement "1" --> "0..*" AuditEvent : tracking
TechnicalActivity "1" --> "0..*" AuditEvent : tracking
TechnicalActivity "*" --> "*" CatalogReference : catalogos FK
AppUser "*" --> "*" TechnicalActivity : responsable
```
